# EraseA12 应用图标设计

**日期：** 2026-07-08
**状态：** 待实施

## 目标

为原生 macOS 应用 EraseA12 设计并接入一套专用应用图标，取代当前因缺少 `AppIcon` 资产而显示的
系统通用图标。图标需要在 Finder、Dock、应用切换器和“关于 EraseA12”窗口中清楚表达“擦除设备”
这一核心能力，并与现有深色玻璃界面和蓝色强调色保持一致。

## 已确认方向

采用“设备轮廓 + 清除光带”方案：一个简化的移动设备轮廓被横向电光蓝清除光带扫过。被扫过的区域
逐渐变为空白或干净的深色表面，让用户直观看出这是擦除工具，而不是普通的设备连接、重启或恢复工具。

## 视觉设计

- 外形：深石墨色圆角方形底板，轮廓简洁、重心居中，符合 macOS 应用图标的体量感。
- 主体：银白色移动设备轮廓，占图标主体约 56%，不画 Apple 标志、实体按键或具体机型细节。
- 动作：一条从左向右扫过设备中部的电光蓝清除光带；光带前后要有明确的“未清除 / 已清除”差异。
- 警示：使用一个很小的暖红色状态点作为不可逆操作提示，不使用大面积红色，避免图标显得像报错工具。
- 配色：石墨黑、银白、电光蓝为主，暖红只作点缀；颜色与应用内 `systemBlue`、深色玻璃背景协调。
- 风格：扁平、精致、略带材质层次；依靠形状和明暗形成识别度，不依赖复杂纹理。
- 构图：主图形四周保留安全边距；在 16×16 像素下仍能看出“设备 + 横向清除动作”。

## 明确避免

- 不出现 `EraseA12`、`A12` 或其他文字，避免小尺寸不可读。
- 不使用 Apple 标志、芯片商标或真实 iPhone 外观细节。
- 不使用垃圾桶、骷髅、爆炸、闪电等过度危险或廉价的视觉隐喻。
- 不使用环形箭头作为主视觉，避免被误解为重启、刷新或恢复。
- 不堆叠 USB 插头、DFU 字样、芯片和手机等多个符号。
- 不使用摄影场景、展示样机、水印或图标之外的背景元素。

## 资产规格

先生成一张 1024×1024 的无文字母版，再从同一母版等比缩放生成标准 macOS `AppIcon.appiconset`：

- 16×16：1x、2x
- 32×32：1x、2x
- 128×128：1x、2x
- 256×256：1x、2x
- 512×512：1x、2x

所有尺寸使用 PNG。母版和最终资产保存在
`EraseA12/EraseA12/Resources/Assets.xcassets/AppIcon.appiconset/`，不把临时生成图、预览图或截图提交到仓库。

## 工程接入

- 新增 `AppIcon.appiconset/Contents.json`，完整声明 10 个 macOS 图标槽位。
- 在 `EraseA12/project.yml` 设置 `ASSETCATALOG_COMPILER_APPICON_NAME: AppIcon`；不把被忽略的
  `EraseA12.xcodeproj` 当作长期配置源。
- 必要时从 `project.yml` 重新生成 Xcode 工程，确保命令行构建实际编译 AppIcon 资产。
- “关于 EraseA12”继续读取 `NSApplication.shared.applicationIconImage`，无需复制另一份图标资源。
- 不改变擦除流程、设备检测、菜单或窗口行为。

## 测试与验收

- 先添加失败检查，证明工程配置声明 `AppIcon` 且资产目录包含全部 10 个槽位。
- 验证所有 PNG 存在、像素尺寸正确、可由系统图像解码器读取。
- 分别检查 16、32、128、512 和 1024 像素效果：小尺寸轮廓不能糊成色块，红色状态点不能抢夺主视觉。
- 运行完整 XCTest，确保图标接入不影响已有 32 项测试。
- 执行 clean Release 构建，并确认 App bundle 包含编译后的图标资源和图标元数据。
- 启动 Release 应用，实际检查 Dock、主窗口及关于窗口中的图标。
- 执行严格签名、universal 架构、依赖、字符串表和 `git diff --check` 验证。
- 不连接设备，不执行任何真机擦除操作。
